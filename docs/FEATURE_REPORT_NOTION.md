## 구현 완료 기능

### WiFi CSI 센싱 (핵심)

| 기능 | 설명 | 결과 |
| --- | --- | --- |
| WiFi CSI 수집 | ESP32가 WiFi 신호의 미세한 변화를 0.05초마다 측정해서 서버로 전송합니다 | 6대 가동중 |
| 호흡 감지 | 사람이 숨 쉴 때 가슴이 움직이면 WiFi 신호가 미세하게 변합니다. 이 변화 주기를 분석해서 분당 호흡수를 알아냅니다 | 14-27 BPM |
| 심박 감지 | 호흡보다 더 미세한 WiFi 위상 변화에서 심박수를 추출합니다. 0.8-2.0Hz 대역 필터링 + 스펙트럼 분석을 사용합니다 | 70-90 BPM |
| 재실 감지 | 방에 사람이 있으면 WiFi 신호 패턴이 바뀝니다. 60초간 빈 방 기준을 학습한 후, 변화가 감지되면 사람 있음으로 판단합니다 | 4개 존 동작 |
| 다중 인원 분리 | WiFi의 30개 서브캐리어(주파수 채널)가 각각 다른 사람에게 반응합니다. 비슷하게 반응하는 채널끼리 묶어서 몇 명인지 추정합니다 | 1-5명 분리 |
| 채널 호핑 | WiFi 채널 1번, 6번, 11번을 50ms마다 번갈아 측정합니다. 하나만 보는 것보다 3배 넓은 주파수 정보를 얻어 정밀도가 올라갑니다 | 3채널 순환 |
| 자세 분류 CSI | WiFi 움직임 크기로 자세를 구분합니다. 움직임 거의 없음은 앉음, 약간이면 서있음, 많으면 걸음, 급격한 변화는 쓰러짐 | 4종 분류 |

### SOTA 신호처리 알고리즘

| 기능 | 설명 | 참조 |
| --- | --- | --- |
| Hampel 필터 | WiFi 신호에서 갑자기 튀는 이상치를 찾아서 자동으로 정상 값으로 교체합니다. 일반 평균 필터보다 50% 더 정확합니다 | wifi-densepose-signal/hampel.rs |
| CSI Ratio | WiFi 칩의 하드웨어 오차(위상 오프셋)를 수학적으로 제거합니다. 기준 서브캐리어와의 비율을 구해서 순수한 환경 변화만 추출합니다 | SpotFi (SIGCOMM 2015) |
| Fresnel Zone | WiFi 송수신기 사이의 물리적 경로를 모델링합니다. 사람의 가슴 움직임(3-15mm)이 이 경로에 얼마나 영향을 주는지 계산해서 호흡 감지 신뢰도를 판단합니다 | FarSense (MobiCom 2019) |
| CSI Spectrogram | WiFi 신호를 시간-주파수 2D 맵으로 변환합니다. 이를 통해 사람의 이동 속도(도플러 효과)를 추정할 수 있습니다 | STFT 기반 분석 |
| Welford 통계 | 메모리를 거의 안 쓰면서 실시간으로 평균/분산/표준편차를 계산합니다. 빈 방 기준선(baseline) 자동 학습에 사용됩니다 | 온라인 알고리즘 |
| 위상 언래핑 | WiFi 위상이 360도를 넘어갈 때 생기는 급격한 점프를 제거합니다. 이걸 안 하면 호흡/심박 추출이 깨집니다 | edge_processing.c |

### 카메라 + AI

| 기능 | 설명 | 결과 |
| --- | --- | --- |
| YOLOv8-pose 감지 | AI가 카메라 영상에서 사람을 찾고, 17개 관절(머리, 어깨, 팔꿈치, 손목, 엉덩이, 무릎, 발목)의 위치를 실시간으로 추적합니다 | 30fps 동작 |
| 스켈레톤 오버레이 | 감지된 관절을 빨간 점으로, 뼈 연결을 노란 선으로 카메라 영상 위에 그립니다 | 실시간 렌더링 |
| 자세 분류 카메라 | 관절 위치 관계로 자세를 판단합니다. 무릎이 엉덩이와 같은 높이면 앉음, 정상 수직이면 서있음, 다리 벌어지면 걸음 | 4종 분류 |
| 디지털 줌 | 카메라 영상을 소프트웨어로 1-5배 확대합니다. 멀리 있는 사람도 크게 볼 수 있습니다 | API 제어 |
| 물체 감지 | 사람뿐만 아니라 모니터, 의자 등 사무실 물체도 감지합니다 | 다중 클래스 |

### 센서 융합

| 기능 | 설명 | 결과 |
| --- | --- | --- |
| 자세 융합 | 카메라와 WiFi CSI가 각각 자세를 판단한 후, 카메라 80% + CSI 20% 비율로 결합합니다. 둘이 같으면 신뢰도 95%로 올라갑니다 | 신뢰도 0.95 확인 |
| 인원수 융합 | WiFi 센서 인원수와 카메라 인원수 중 높은 값을 채택합니다. WiFi는 벽 뒤도 감지하고, 카메라는 시야 내에서 정확합니다 | max 방식 |
| 4개 존 관리 | 6대 노드가 4개 호실에 분산 배치되어 각 호실별로 독립적으로 인원과 상태를 집계합니다 | per-zone 집계 |

| 센싱 페이지 | 노드별 RSSI 스파크라인, 모션/호흡/심박 바, 자세 인디케이터를 전용 페이지에서 상세하게 보여줍니다 | /sensing 라우트 |

## 시스템 아키텍처

```
ESP32-S3 x 6대                    서버 (사내망)
+--------------+            +----------------------+
| WiFi CSI     |--UDP 20Hz->| signal-adapter :8001 |
| ch1/6/11     |            |  +- CSI 신호처리      |
| 호흡/심박/    |            |  +- 재실/자세 감지    |
| 모션/낙상     |            |  +- Welford z-score  |
+--------------+            |  +- WS broadcast     |
                            +----------------------+
USB 카메라                   | camera-service :8002 |
```
