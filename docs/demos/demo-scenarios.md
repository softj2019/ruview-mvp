## 시나리오 1: 시스템 개요 시연 (2분)

### 목적
비전문가가 "이 시스템이 뭘 하는지" 즉시 이해

### 시연 순서
1. 브라우저에서 대시보드 접속
2. 상단 KPI 카드 4개 설명 (Online Devices, Active Zones, Presence, Alerts)
3. 좌측 2D Floor Plan 설명 — 장치 위치, 존 구분, 커버리지
4. 우측 3D Observatory 미니뷰 설명 — 전체화면 전환 시연
5. 하단 이벤트/디바이스/신호 패널 설명

### 핵심 메시지
> "카메라 없이, 웨어러블 없이, Wi-Fi 신호만으로 사람의 존재, 움직임, 낙상을 실시간 감지합니다."

### 체크리스트
- [ ] 대시보드 페이지 정상 로딩
- [ ] KPI 카드 4개 표시
- [ ] 2D Floor Plan 렌더링
- [ ] 3D Observatory iframe 로딩
- [ ] 네비게이션 메뉴 동작

## 시나리오 2: 재실 감지 시연 (2분)

### 목적
사람이 공간에 들어왔을 때 실시간 감지되는 것을 보여줌

### 시연 순서
1. Settings 페이지에서 시나리오를 "idle"로 설정
2. Dashboard 확인 — 이벤트 없음, 조용한 상태
3. Settings에서 시나리오를 "presence"로 변경
4. Dashboard로 돌아와 변화 확인:
   - KPI "Presence Count" 증가
   - AlertPanel에 "presence detected" 이벤트 표시
   - FloorView에서 존 색상 변화 (inactive → active)
   - SignalChart에서 신호 변화

### 핵심 메시지
> "누군가 방에 들어오면 즉시 감지됩니다. 응답 시간 1초 이내."

### 체크리스트
- [ ] idle → presence 전환 성공
- [ ] presence_detected 이벤트 생성
- [ ] KPI 수치 변경
- [ ] 존 상태 변경 시각화
- [ ] 신호 그래프 변화

### 시연 순서

### 핵심 메시지

### 체크리스트
- [ ] motion_active 이벤트 생성
- [ ] 신호 진폭 변화 시각화
- [ ] idle/presence와의 차이 명확

## 시나리오 4: 낙상 감지 시연 (2분) ⭐ 핵심

### 목적
가장 중요한 안전 기능 — 낙상 발생 시 즉각 알림

### 시연 순서
1. 시나리오를 "fall"로 변경
2. Dashboard에서 즉시 변화:
   - **fall_suspected** 이벤트 (critical severity)
   - KPI "Critical Alerts" 카운터 증가 (빨간색)
   - AlertPanel에 빨간색 경고 표시
   - 신호에서 급격한 스파이크 후 드롭 패턴
3. Events 페이지로 이동하여 상세 기록 확인
4. 이벤트 테이블에서 confidence, timestamp, device 정보

### 핵심 메시지
> "독거노인이 넘어지면 즉시 관리자에게 알림이 갑니다. 골든타임 확보의 핵심입니다."

### 체크리스트
- [ ] fall_suspected 이벤트 즉시 생성
- [ ] severity: critical 표시
- [ ] KPI 빨간색 카운터 증가
- [ ] 신호 스파이크+드롭 패턴 시각화
- [ ] Events 페이지에서 상세 기록 확인

## 시나리오 5: 생체신호 모니터링 시연 (1분)

### 목적
호흡/심박 감지 가능성 시연 (향후 고도화 방향)

### 시연 순서
1. 시나리오를 "breathing"으로 변경
2. SignalChart에서 주기적 패턴 확인
3. 향후 바이탈 사인 패널 추가 계획 설명

### 핵심 메시지
> "Wi-Fi 신호로 호흡률과 심박수까지 추정할 수 있습니다. 수면 모니터링에 활용 가능합니다."

### 체크리스트
- [ ] breathing 시나리오에서 주기적 신호 패턴
- [ ] 15 BPM 호흡 + 72 BPM 심박 modulation 확인
